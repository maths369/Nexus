# Knowledge Capture

在用户要把想法、结论、研究笔记、战略想法沉淀进知识库时，优先使用这个 skill。

## 目标

把信息组织成后续可扩展、可链接、可检索的知识页面，而不是一次性文本输出。

## 使用原则

1. 优先创建页面并沉淀到 Vault。
2. 优先使用结构化 section，而不是大段无结构文本。
3. 需要跨页关联时，插入 page link。
4. 如果内容天然是一个追踪容器，考虑创建 database 页面。

## 常用工具

- `create_note`
- `document_append_block`
- `document_replace_section`
- `document_insert_page_link`
- `document_create_database`

## 推荐页面结构

1. `## 背景`
2. `## 关键观点`
3. `## 决策/结论`
4. `## 后续行动`
5. `## 参考`
